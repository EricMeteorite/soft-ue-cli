# soft-ue-cli 零基础开始指南

这份文档是给第一次接触这个项目的人写的。

你可以把它理解成一句话：

soft-ue-cli 是一个放在这个文件夹里的小工具，它自己本身可以完全只在这个文件夹里运行；但如果你想让它真的去控制 Unreal Engine，那么你的 UE 项目里必须临时接入一个叫 SoftUEBridge 的项目级插件。

这不是隐藏条件，而是这个项目的工作原理。

## 1. 先说最重要的结论

这套方案里，一共有两层东西：

1. 这个仓库里的 Python 工具。
2. UE 项目里的 SoftUEBridge 插件。

它们分别负责不同的事：

1. Python 工具负责你在命令行里输入的命令。
2. UE 插件负责在 Unreal Editor 里面真正执行这些命令。

所以：

1. 只部署 Python 工具，不会影响系统环境，也不会影响引擎和项目。
2. 但只部署 Python 工具，还不能控制 UE。
3. 想控制 UE，就必须把插件接到某个 UE 项目里。
4. 这个接入可以做到只改项目、不改引擎、而且可以卸载回退。

## 2. 你现在文件夹里有哪些傻瓜入口

你可以直接双击下面这个文件：

[START_HERE.bat](START_HERE.bat)

它会弹出一个菜单，让你选要做什么。

如果你不想记命令，也完全可以只用这个 bat。

## 2.5 这个项目到底是怎么运行的

这部分我用最白话的方式讲。

你可以把整个东西想象成两个人在配合干活：

1. 这个文件夹里的 soft-ue-cli，负责听你下命令。
2. UE 里的 SoftUEBridge 插件，负责替你在编辑器里真正动手。

它们的关系像这样：

1. 你在 bat 菜单里选一个功能，或者你手动运行 soft-ue-cli 命令。
2. soft-ue-cli 这个 Python 工具把命令变成一个本地 HTTP 请求。
3. UE 项目里的 SoftUEBridge 插件收到这个请求。
4. 插件在 Unreal Editor 里执行真正的操作，比如查关卡对象、生成 Actor、读取日志。
5. 插件把结果再回给 soft-ue-cli。
6. 你在终端里看到结果。

所以这里一定有两个部分：

1. 命令发送端，也就是当前这个仓库里的工具。
2. 命令执行端，也就是 UE 项目里的插件。

如果只有前者，没有后者，会发生什么？

答案是：

1. 工具本体可以正常启动。
2. 帮助信息可以正常显示。
3. 但是它连不到 UE，因为 UE 里根本没有桥接端。

这就是为什么你会看到“还要装插件”这件事。

你也可以直接把它记成下面这句最白的话：

1. 不装插件时，这个仓库里的工具只能“自己正常启动”。
2. 装了插件之后，这个仓库里的工具才能“真的连上 UE 并控制 UE”。

所以插件不是附加小功能，而是 UE 端真正执行命令的那一半。

## 2.6 这个插件到底是什么插件

这个插件的名字叫 SoftUEBridge。

它不是系统插件，不是 Windows 插件，也不是要装到你的 Python 里。

它是一个 Unreal Engine 项目级插件。

它原本放在这个仓库里：

1. soft_ue_cli/plugin_data/SoftUEBridge

当你在菜单里选择“把插件装进某个 UE 项目”时，做的事情其实就是：

1. 把这个 SoftUEBridge 文件夹复制到你的 UE 项目里的 Plugins 目录。
2. 在你的 .uproject 文件里把它启用。

也就是说，这个插件本来就跟着这个仓库一起提供，只是默认还没有接到你的 UE 项目上。

你可以把它理解成：

1. 这个仓库里放着一个“待安装的项目插件”。
2. 只有当你明确选择安装到某个项目时，它才会进入那个项目。

再说得更直接一点：

1. 插件本体已经跟着这个仓库一起提供了。
2. 你不需要去网上额外下载别的插件。
3. 你做的“安装插件到项目”，本质上只是把仓库里已经带着的 SoftUEBridge 复制到你的目标项目里。

## 2.7 它会不会改引擎

不会。

这里走的是项目级插件方案，不是引擎级插件方案。

也就是说：

1. 它不会往你的 UnrealEngine56 目录里复制东西。
2. 它只会改你明确指定的那个项目目录。
3. 后面你也可以把它从项目里卸载掉。

## 3. 每个选项到底是什么意思

双击 [START_HERE.bat](START_HERE.bat) 以后，你会看到一组数字选项。下面是每个选项的白话解释。

### 选项 1：一键准备当前工具

这个选项会做的事情：

1. 在当前 soft-ue-cli 文件夹里创建或复用 .venv。
2. 在这个 .venv 里安装 soft-ue-cli 自己和测试依赖。
3. 自动跑测试，确认工具本体没坏。

这个选项不会做的事情：

1. 不会修改系统 PATH。
2. 不会往系统 Python 里装包。
3. 不会修改 Unreal Engine。
4. 不会修改你的 UE 项目。

什么时候该先点它：

1. 第一次使用时。
2. 你怀疑环境不对的时候。
3. 更新了这个仓库之后，想重新确认工具没问题的时候。

### 选项 2：打开零基础说明文档

这个选项会直接打开这份说明文档和另一份更完整的部署说明。

适合什么时候看：

1. 你还没弄明白自己到底在做什么的时候。
2. 你想知道某个操作到底会改哪里的时候。

### 选项 3：确认当前工具是否安装好

这个选项会做的事情：

1. 调用本地 CLI 输出帮助信息。
2. 让你确认工具本身能启动。

如果这个选项都失败，先不要碰 UE 项目，先把工具本体准备好。

### 选项 4：把插件装进某个 UE 项目

这个选项会先让你输入一个 UE 项目根目录路径。

比如：

```text
D:\Git\Github\UESource\ProjectR
```

然后它会做两件事：

1. 把插件复制到 你的项目/Plugins/SoftUEBridge
2. 修改 你的项目 根目录下的 .uproject，让插件启用

它不会做的事情：

1. 不会修改引擎目录。
2. 不会修改系统环境。
3. 不会修改别的项目。

这一步做完后，你还不能立刻用。你还需要重新生成项目文件并重新编译项目。

### 选项 5：从某个 UE 项目里卸载插件

这个选项也会让你输入 UE 项目根目录路径。

它会做三件事：

1. 删除 项目/Plugins/SoftUEBridge
2. 从 .uproject 里去掉 SoftUEBridge 项
3. 删除 项目/.soft-ue-bridge 这个运行时状态目录

适合什么时候用：

1. 你不用这个工具了。
2. 你想恢复项目到未接入状态。
3. 你想彻底确认项目里没有这个工具留下的项目级痕迹。

### 选项 6：检查某个项目有没有接好

这个选项会检查三件事：

1. 插件文件在不在。
2. .uproject 里有没有启用插件。
3. 运行中的 UE 里，桥接服务能不能连上。

如果前两项通过、第三项失败，通常表示：

1. 你还没启动编辑器。
2. 或者启动了，但项目还没成功编译插件。

### 选项 7：查看某个项目当前的桥接状态

这个选项会进入你指定的项目目录，然后执行状态检查。

它适合你在编辑器已经启动后快速确认：

1. 当前桥是否在线。
2. 当前项目的桥接端口是否被自动识别。

### 选项 8：查看某个项目当前关卡里的对象

这个选项会进入你指定的项目目录，然后执行最常见的只读查询命令。

它适合当作“第一次真正连通 UE”的测试。

如果它能返回对象列表，说明你的工具链已经基本通了。

## 4. 作为小白，你最安全的使用顺序

请按这个顺序走，不要跳：

1. 双击 [START_HERE.bat](START_HERE.bat)
2. 先选 1，准备当前工具
3. 再选 3，确认工具本体可以启动
4. 再选 2，看一下文档
5. 如果你决定真的接入 UE 项目，再选 4
6. 然后去重新生成并编译你的 UE 项目
7. 启动 Unreal Editor
8. 回到 bat 菜单，选 6 检查项目是否接好
9. 再选 7 看桥接状态
10. 最后选 8 看对象列表

先读，后写，最后再做复杂操作。这是最稳妥的顺序。

## 5. 你最容易误解的地方

### 误解 1：我以为一键部署后就能直接控制 UE

不是。

一键准备当前工具，只是把 Python 工具准备好。

如果你还没把插件接入 UE 项目，工具虽然能启动，但它控制不了 UE。

### 误解 2：我不想碰项目，但我又想控制项目

这两个要求不能同时成立。

原因不是我故意为难你，而是技术上就是这样：

1. 没有项目里的插件，UE 进程里就没有桥。
2. 没有桥，CLI 就没有东西可以连接。

### 误解 3：安装插件是不是会改引擎

不会。

我们这里走的是项目级插件方案，只会改你指定的那个 UE 项目目录。

## 6. 什么时候说明已经成功了

你至少要看到下面这些现象，才算真的成功：

1. 选项 1 成功跑完。
2. 选项 3 能正常输出帮助。
3. 选项 4 已经把插件装到项目里。
4. 项目已经重新生成并编译成功。
5. Unreal Editor 已经启动。
6. 选项 6 里三项检查都通过。
7. 选项 8 能返回当前关卡对象列表。

## 7. 失败时先看哪里

如果出问题，先按这个顺序排查：

1. 先看是不是没先做选项 1。
2. 再看项目路径是不是输错了。
3. 再看你的项目根目录下是不是真的有 .uproject。
4. 再看插件是不是已经复制到 项目/Plugins/SoftUEBridge。
5. 再看 .uproject 里是不是启用了 SoftUEBridge。
6. 再看项目有没有重新生成和重新编译。
7. 再看 Unreal Editor 是不是已经启动到项目主界面。
